🧠 Ontology Overview

  • Purpose & Scope
     The Visual Arts DNA model represents the creative process of visual arts—from conceptual “idea” to executed work—capturing roles, stages, materials, actions, and outcomes. It’s designed as a formally structured ontology (likely OWL/RDF), aiming to foster interdisciplinary clarity and interoperability.
  • Core Structure
  • Entities: Includes agents (artists, collaborators), objects (tools, media), steps (e.g., ideation, sketching, fabrication), and artifacts (finished artworks, drafts).
  • Relationships: Defines typical links like performsAction, usesMaterial, resultsIn, influencesBy, etc.
  • Contextual Extensions: Includes metadata about time, place, influences, and socio‑cultural context—echoing CIDOC‑CRM’s handling of provenance and contextual info philarchive.org+1digitalartarchive.at+1research.vu.nlmdpi.com.
  • Methodological Influences
     The model draws inspiration from existing ontologies like CIDOC‑CRM and the Function‑Behavior‑Structure ontologies used in design modeling en.wikipedia.org. It also integrates situational layers—reflecting the ‘Descriptions & Situations’ framework (D&S) for contextual interpretation mdpi.com.
  • Key Features
  • Ontology-driven abstraction: Enables formal querying—“find artworks where 'materia' influenced tool selection”—and supports cross‑collection interoperability.
  • Descriptor-level identity: Defines atoms like “brushstroke type X” as namable entities, enabling consistent tracking across works.
  • Provenance & relationships: Tracks influences, roles, and processes akin to historical ontologies like HiCO and PHAROS .
  • Representation‐agnostic: Designed to accommodate multiple interpretations—e.g., the same iconographic subject tagged differently—with contextual situational qualifiers mdpi.com.
  • Strengths & Limitations
  • Strengths: Granular contextual modeling of materials, processes, roles; historically informed via rich art‑heritage ontologies.
  • Gaps: Semantic depth for visual/formal attributes (shape, color, gesture) is limited—difficult to do image-based retrieval or compare formal features computationally. Also, contextual annotation appears to depend on curator input and isn’t fully automated.

Overview of the Ontology and Thesaurus

The Visual Arts DNA (VADA) ontology and thesaurus aim to provide a structured framework for modeling the visual arts creative process domain. The ontology defines the concepts, relationships, and rules that govern the domain, while the thesaurus provides a controlled vocabulary for describing and annotating visual arts-related data.

Ontology Analysis

Key Concepts

The VADA ontology includes several key concepts that represent key entities in the visual arts domain:

  1. Artwork: represents a creative work, such as a painting, sculpture, or installation.
  2. Artist: represents an individual who creates artworks.
  3. Style: represents a distinct approach or technique used in creating artworks.
  4. Movement: represents a group of artists who share similar styles, themes, or philosophies.
  5. Medium: represents the material or technique used to create an artwork.

Relationships

The ontology defines several relationships between these concepts:

  1. createdBy: links an artwork to its creator (artist).
  2. hasStyle: links an artwork to its associated style.
  3. partOf: links an artwork to a movement or collection.
  4. usesMedium: links an artwork to its medium.

Inference Rules

The ontology includes inference rules that enable reasoning and automatic generation of new knowledge:

  1. Transitive relationships: if artwork A is part of movement B, and movement B is part of style C, then artwork A is also part of style C.
  2. Inverse relationships: if artist A created artwork B, then artwork B was created by artist A.

Thesaurus Analysis

The VADA thesaurus provides a controlled vocabulary for describing and annotating visual arts-related data. It includes several key features:

  1. Hierarchical structure: terms are organized in a hierarchical structure, with broader terms subsuming narrower terms.

  3. Definitions and scope notes: each term includes a definition and scope note, providing context and guidance for usage.

Review and Analysis of the VisualArtsDNA Ontology and Thesaurus

Overview

The VisualArtsDNA project provides two core knowledge structures for modeling the visual arts creative process domain:

  • An ontology, expressed in OWL, available at http://visualartsdna.org/model13.
  • A controlled vocabulary (thesaurus), represented in SKOS, available at http://visualartsdna.org/vocab23.

These resources are designed to work together, providing both a formal conceptual framework and a rich, hierarchical vocabulary for describing and analyzing the visual arts domain.

Ontology: Information Model for the Visual Arts

Structure and Purpose

  • The ontology is titled "Information Model for the Visual Arts" and is implemented in OWL (Web Ontology Language)3.
  • It organizes details of the visual arts creative process into classes and relationships, capturing the domain’s structure in a machine-readable, semantic format3.
  • The ontology is not presented as definitive or complete, suggesting it is intended as a flexible, extensible foundation for further development3.

Key Features

  • Defines classes representing key entities in the visual arts domain (e.g., Artist, Artwork, Technique, Material).
  • Uses rdfs:subClassOf to establish class hierarchies, enabling inheritance and specialization of concepts3.
  • Designed to be queried and browsed, with support for SPARQL queries and instance data exploration3.

Thesaurus: Visual Arts Vocabulary

Structure and Purpose

  • The thesaurus is provided in SKOS (Simple Knowledge Organization System), a standard for representing controlled vocabularies and taxonomies3.
  • It is available both as a graphical visualization and as an RDF (TTL/text) file3.
  • The vocabulary covers visual arts terms and is organized hierarchically using skos:broader relationships3.

Key Features

  • Terms are grouped into subject areas or modules using skos:inScheme, reflecting different facets of the visual arts domain3.
  • The hierarchical structure allows for navigation from broader to narrower concepts, supporting both generalization and specialization3.

Integration and Semantic Alignment

Relationship Between Ontology and Thesaurus

  • The ontology and thesaurus use different knowledge representation formalisms (OWL for the ontology, SKOS for the thesaurus), but are designed to be semantically aligned3.
  • Concepts in the thesaurus serve as a vocabulary for populating or describing instances of the classes defined in the ontology3.
  • Hierarchical relationships in the thesaurus (skos:broader) correspond to class hierarchies in the ontology (rdfs:subClassOf), ensuring coherence between the two structures3.

Functionality

  • The thesaurus provides the domain-specific terms needed for detailed annotation and description, while the ontology provides the structural and relational framework for organizing these terms3.
  • Together, they enable a rich, structured, and semantically consistent representation of knowledge in the visual arts creative process domain3.
